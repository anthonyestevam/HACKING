O OPEN REDIRECT É UMA FALHA DE REDIRECIONAMENTO, MUITOS SITES UTILIZAM O REDIRECIONAMENTO DE UMA PÁGINA PARA OUTRA SÓ QUE NEM SEMPRE 
ELAS SÃO TRATADAS, PODENDO ASSIM HAVER UMA FALHA DE OPEN REDIRECT. COM ESSA VULNERABILIDADE É POSSÍVEL DE ALGUM USUÁRIO CAIR EM UM 
PHISHING. A FALHA É ENCONTRADA EM URLs QUE PEDEM REDIRECIONAMENTO PARA OUTRA PÁGINA UTILIZANDO CÓDIGOS COMO 'redirect=' OU 'next='.
	
		EX: https://site.com/login?redirect=https://site.com
			
		   https://site.com/login?next=https://site.com
		  
	1.CHECAR SE A URL TEM OS PARÂMETROS CITADOS ANTERIORMENTE OU PARECIDOS.
	
	2.ALTERAR A URL QUE VEM DEPOIS DO PARÂMETRO.
	
		COMO POR EXEMPLO ALTERAR: https://site.com/login?next=https://site.com
		
		PARA: https://site.com/login?next=https://instagram.com
	
	3.CASO O SITE SEJA REDIRECIONADO PARA O OUTRO, SIGNIFICA QUE A FALHA ESTÁ EXISTENTE.
	
OBS: ESSES REDIRECIONAMENTOS NORMALMENTE OCORREM QUANDO SE ENTRA EM UMA PÁGINA DE LOGIN PARA DEPOIS IR PARA HOME OU OUTRA ÁREA.

	EXTRA: PODE SE FAZER UM XSS PELO OPEN REDIRECT MUDANDO O URL MALICIOSO PARA javascript:alert(1).
		
		EX: ttps://site.com/login?next=javascript:alert('xss')

	DICA: CASO O SITE TENHA ÁREA DE LOGIN.
		
		1.CRIE UMA CONTA
		
		2.ACESSE A CONTA
		
		3.COPIE A URL
		
		4.SAIA DA CONTA
		
		5.COLE A URL NA BARRA DE NAVEGAÇÃO.
		
		6.ANALISE QUE PROVAVELMENTE A URL TERÁ UM PARÂMETRO
		DE REDIRECIONAMENTO, USE ELE PARA TESTE.
