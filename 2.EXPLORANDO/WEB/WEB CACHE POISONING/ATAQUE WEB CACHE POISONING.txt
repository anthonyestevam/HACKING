COMO EXPLORAR UMA VULNERABILIDADE DE WEB CACHE POISONING.
	
	1.DESCOBRIR SE O SERVIDOR ARMAZENA O CACHE
	
	2.SE ARMAZENAR, INTERCEPTE A REQUISIÇÃO QUE ARMAZENA 
	
	3.ENVIE A REQUISIÇÃO PARA O PARAM MINER.
	
	4.VÁ EM EXTENSIONS > PARAM MINER > OUTPUT
	
	5.NO OUTPUT MOSTRARÁ OS HEADERS QUE ESTÃO ALTERANDO
	A RESPOSTA DA REQUISIÇÃO.
	
	6.ENVIE A REQUISIÇÃO PARA O REPEATER.
	
	7.USE OS HEADERS QUE APARECERAM NO PARAM MINER NA REQUISIÇÃO
	E ANALISE COMO O SERVIDOR RESPONDE Á ESSE HEADER E SE A RESPOSTA
	DELES ESTÁ SENDO DEVIDAMENTE HIGIENIZADA E SE ESTÁ GERANDO DADOS
	DINÂMICOS.
	
	8.CASO PERCEBA QUE ESSE HEADER UTILIZADO NÃO ESTÁ SENDO HIGIENIZADO
	CORRETAMENTE OU GERANDO DADOS DINÂMICOS, ESSE HEADER VULNERÁVEL QUE SERÁ
	UTILIZADO PARA O ATAQUE.
	
	9.TENTE USAR PAYLOADS NO HEADER VULNERÁVEL COMO PAYLOADS DE XSS OU UM OPEN 
	REDIRECT E ANALISE COMO O SERVIDOR RESPONDE.
	
	10.VOLTE A REQUISIÇÃO INTERCEPTADA E INJETE O HEADER COM O PAYLOAD
	NELE.
	
	11.CLIQUE EM 'FORWARD'.
	
	12.DESINTERCEPTE E VOLTE PARA A PÁGINA, SE ELA RESPONDER AO PAYLOAD
	SIGNIFICA QUE O ATAQUE FOI FEITO.
