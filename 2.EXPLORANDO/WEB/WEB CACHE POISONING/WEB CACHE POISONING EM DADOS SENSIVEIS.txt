O WEB CACHE POISONINHG PODE SER UTILIZADO PARA ROUBAR INFORMAÇÕES SENSIVÉIS
COMO TOKEN DE SESSÃO POR EXEMPLO.
	
	1.CRIE UMA CONTA E LOGUE NO SITE
	
	2.VÁ EM TARGET
	
	3.ANALISE AS REPOSTAS DAS REQUISIÇÕES PARA
	VER SE ELAS TEM DADOS IMPORTANTES COMO O TOKEN DE SESSÃO 
	OU OUTRAS INFORMAÇÕES IMPORTANTES.
	
	4.ENVIE A REQUISIÇÃO ENCONTRADA PARA O REPEATER.
	
	5.ADICIONE UMA EXTENSÃO COMO .JS OU .CSS NO FINAL DO ENDPOINT.
	
		EX: GET /search/ HTTP/2
		
		ALTERE PARA: GET /search/.js HTTP/2
		
		
	6.ENVIE A REQUISIÇÃO E VEJA SE O SERVIDOR RESPONDE COM CÓDIGO 200.
		
		CASO O SERVIDOR NÃO RESPONDA COM 200, TENTE USAR UM ';' ANTES
		DA EXTENSÃO E ENVIE NOVAMENTE PARA VER SE O SERVIDOR RESPONDE COM 200.
			
			EX: GET /search/;.js HTTP/2
	
	7.SE DER RESPOSTA 200, CLIQUE COM O BOTÃO DIREITO ENCIMA DA REQUISIÇÃO
	E CLIQUE EM 'COPY URL'.
	
	8.COLOQUE A URL NO NO NAVEGADOR E VEJA SE CONSEGUIU SER ACESSADO OU APARECEU
	UMA PÁGINA EM BRANCO.
	
	9.CASO TENHA OCORRIDO A FASE 8, ENVIE A URL PELO TERMINAL USANDO O COMANDO curl.
		
		EX: curl 'https://site.com/search/.js' --compressed
		
	10.VEJA SE A INFORMAÇÃO CONFIDENCIAL QUE TINHA NA RESPOSTA DO REPEATER, TAMBÉM
	APARECEU NA RESPOSTA DO CURL, SE TIVER APARECIDO SIGNIFICA QUE O ATAQUE FOI BEM
	OCORRIDO.
	
	ESSE ATAQUE É INTERESSANTE POIS PODE FAZER UM ACCOUNT TAKEOVER CASO CONSIGA ROUBAR DADOS
	COMO UM TOKEN DE SESSÃO.
	
	EXTRA: VOCÊ PODE USARR O GREP QUANDO FOR FAZER O USO DO CURL PARA PODER VER A INFORMAÇÃO
	COM MAIS EXATIDÃO.
		
		EX: NA RESPOSTA DA REQUISIÇÃO TEM O PARÂMETRO 'cookie' COM O VALOR DENTRO DELE.
		
		USE NO CURL: curl 'https://site.com/search/.js' --compressed | grep 'cookie'
		
		ASSIM IRÁ FILTRAR A RESPOSTA DO SERVIDOR MOSTRANDO O COOKIE LOGO NO INICIO DA RESPOSTA,
		OTIMIZANDO SEU TEMPO.
