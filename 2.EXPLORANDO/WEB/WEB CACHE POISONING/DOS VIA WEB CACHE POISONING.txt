EM ALGUNS SERVIDORES COMO O DO AKAMAI-CDN, É POSSÍVEL
DERRUBAR O SERVIDOR COM WEB CACHE POISONING, FAZENDO COM QUE OS 
CLIENTES NÃO CONSIGAM ACESSAR A PÁGINA ATACADA.
	
	1.DESCOBRIR SE A RESPOSTA DA REQUISIÇÃO É ARMAZENADA EM CACHE.
	
	2.ENVIE A REQUISIÇÃO PARA O REPEATER.
	
	3.ADICIONE UMA CONTRA BARRA COMO SE FOSSE UM HEADER NA REQUISIÇÃO.
		
		EX: GET / HTTP/2
		    Host: site.com
		    \:
		    
	4.ENVIE A REQUISIÇÃO E VEJA SE O SERVIDOR RESPONDE COM CÓDIGO 400 E
	SE FOI ARMAZENADO EM CACHE.
	
	5.SE RESPONDER COM CÓDIGO 400, CLIQUE COM O BOTÂO DIREITO ENCIMA DA
	REQUISIÇÃO, CLIQUE EM 'COPY URL'.
	
	6.COLOQUE A URL NO NAVEGADOR E VEJA SE QUANDO RODAR O SITE TAMBÉM DARÁ ERRO 400.
	
	7.SE DER ERRO 400 PELO NAVEGADOR TAMBÉM, SIGNIFICA QUE O ATAQUE FOI FEITO.
	
	EXTRA: APÓS DAR CÓDIGO 400, COLOQUE RAPIDAMENTE A URL NO NAVEGADOR E VEJA,
	POIS ALGUNS SERVIDORES PODEM ARMAZENAR O CACHE POR VOLTA DE 1 SEGUNDO, 
	ENTÃO A ANÁLISE PRECISA SER FEITA RÁPIDO.
