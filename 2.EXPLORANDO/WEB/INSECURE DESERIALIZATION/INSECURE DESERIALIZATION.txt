O INSECURE DESERIALIZATION É UMA FALHA DE DESSERIALIZAÇÃO INSEGURA DO SITE.
 	
 	O QUE É SERIALIZAÇÃO E DESSERIALIZAÇÃO ?
 	
 		1.SERIALIZAÇÃO: CODIFICAR UM OBJETO EM BINARIO, BASE64 ETC... E SALVAR NO BANCO DE 
 		DADOS.
 		
 		2.DESSERIALIZAÇÃO: TRANSFORMAR ESSA CODIFICAÇÃO EM OBJETO NOVAMENTE.
 		
 O QUE OCORRE É QUE PODE OCORRER DE VOCÊ DESSERIALIZAR ESSE CÓDIGO E TER INFORMAÇÕES QUE VOCÊ PODE
 ALTERAR E USA-LA NOVAMENTE PODENDO FAZER ESCALAÇÃO DE PREVILEGIOS, MUDAR PARA OUTRA CONTA PODENDO EXECUTAR
 CÓDIGOS.
 
 	EX:
 		O COOKIE:  
                         Tzo0OiJBdXRoIjozOntzOjU6InVzZXIiO3M6NToiamFtZXMiO3M6MzoibG9naW4iO3M6ODoiaXNsb2dnZWQiO3M6MjoiaWQiO2I6MTt9
                         
                PODE SER DECODIFICADO JÁ QUE É UM BASE64 COM UM DECODIFICADOR ON-LINE NO NAVEGADOR OU POR UMA FERRAMENTA.
                
                COOKIE DECODIFICADO:
                			O:4:”Auth”:3:{s:5:”user”;s:5:”james”;s:3:”login”;s:8:”islogged”;s:2:”id”;b:1;}
                			
                			O = REPRESENTA UM OBJETO
                			S= STRING
                			B = VALOR BOOLEANO
                			
                ALTERANDO O VALOR BOOLEANO DE 1 PARA 0, CODIFICANDO ELE PARA BASE64 NOVAMENTE, COPIA-LO, SUBSTITUIR O COOKIE ANTIGO PELO NOVO E O SERVIDOR PERMITIR. PROVAVELMENTE VOCÊ SERÁ AGORA 
                ADMINISTRADOR DO SITE.
                
                ISSO É A DESSERIALZAÇÃO INSEGURA.
                
	 A DESSERIALIZAÇÃO NÃO PODE SER ENCONTRADA APENAS POR COOKIE, ENTÃO É INTERESSANTE QUE VOCÊ FAÇA UM CODE-REVIEW PPARA ENTENDER COMO FUNCIONA,TAL CODE REVIEW QUE PODE ADVIR DA EXPLORAÇÃO DE OUTRA 	
	 VULNERABILIDADE COMO XXE, LFI, INFORMATION DISCLOSURE, ETC. E DEPENDENDO DA SITUAÇÃO PODERÁ SER EXTRAIDO UMA KEY QUE ESTÁ SENDO USADA PARA ASSINAR UM TOKEN E FAZER O PROCESSO REVERSO DE GERAÇÃO 
	 DO TOKEN COM O SEU PAYLOAD CONTIDO NELE.
	 
	 EXTRA: PODE SE ENCONTRAR UMA INSECURE DESERIALIZATION USANDO BURPSUITE PARA INTERCEPTAR 
	 REQUISIÇÕES QUE PODEM TER SERIALIZAÇÕES INSEGURAS.
