SSRF É UMA VULNERABILIDADE EM QUE VOCÊ ENVIA REQUISIÇÕES COMO SE FOSSE O SERVIDOR QUE ESTIVESSE AS MANDANDO
O SSRF BURLA O FIREWALL PORQUE AS REQUISIÇÕES SERIAM COMO SE FOSSE DO PRÓPRIO SERVIDOR.

MÉTODOS DE ATAQUE:
	
	POR URL:
		CASO O SERVIDOR PEÇA A REQUISIÇÃO PELA URL, A URL SERÁ PARECIDA COM ESSA POR EXEMPLO: https://seusite.com/page.php=https://menu.seusite.com
			
		COM ISSO, TALVEZ ELE POSSA TER VULNERABILIDADE SSRF.
		
			1.PODE SE ACESSAR SITES INTERNOS COMO ADMIN DO SERVIDOR ALTERANDO A REQUISIÇÃO:
		
				EX: https://seusite.com/page.php=http://localhost/admin
		  	 	 
		  	 				 OU
		  	 			 
	        		EX: https://seusite.com/page.php=http://127.0.0.1/admin
	        		
	        	2.PODE ACESSAR ARQUIVOS INTERNOS DO SERVIDOR:
	        	
	        		EX: https://seusite.com/page.php=file:/etc/passwd
	        				
	        					OU
	        		EX: https://seusite.com/page.php=file:///etc/passwd
	        		
	POR MÉTODO POST:
		
		PELO METÓDO POST, SERIA VOCÊ USAR UM PROXY COMO O BURPSUITE.
		
		1.INTERCEPTAR A REQUISIÇÃO
		
		2.CASO A REQUISIÇÃO ESTEJA USANDO O IP DO SERVIDOR. EX: https://127.182.14.23:8080/stock/productid=1
		
		3.ALTERE A REQUISIÇÃO IGUAL SERIA PELA URL PARA ACESSAR SITES INTERNOS OU ARQUIVOS INTERNOS.
			
			EX: https://127.182.14.23:8080/admin
					
		4.ENVIAR A REQUISIÇÃO, CASO O ENVIO DÊ CÓDIGO 200 SIGNIFICA QUE O SSRF DEU CERTO.
		
		5.AGORA SÓ EXPLORAR.
		
	AS VEZES PODE OCORRER DA URL ESTA ENCODADA, PARA ISSO:
	
		1.DESENCODE ELA.
		
		2.ALTERE PARA FORMA QUE VAI USAR.
		
		3.ENCODE NOVAMENTE
		
		4.SUBSTITUA PELA NOVA URL ENCODADA.
		
		OBS: ISSO SERVE TANTO PARA ATAQUE PELA URL, QUANTO PELO METÓDO POST.
