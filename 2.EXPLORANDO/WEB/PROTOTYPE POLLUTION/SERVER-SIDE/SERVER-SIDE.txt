	2.PODE CAUSAR PROBLEMAS DOS.
	
	3.A ALTERAÇÃO PERSISTE DURANTE TODA VIDA ÚTIL DO NODE JS E NÃO PODE
	REDEFINI-LA.
	
 	O PROTOTYPE POLLUTION SERVER-SIDE SÓ PODE OCORRER SE O BACK-END DO APLICATIVO
 	FOR JAVA SCRIPT COMO O NODE JS. 
 	
 	1.PROCURAR REQUISIÇÃO.
 		
 		1.ABRA UM PROXY
 		
 		2.ENVIE REQUISIÇÕES POST
 		
 		3.PROCURE ELA
 		
 		4.VEJA EM 'Content-Type' SE FOI ENVIADA EM FORMATO JSON.
 		
 		5.SE FOR NESSE FORMATO, VAMOS PARA O PRÓXIMO PASSO.
 	
 	2.IDENTIFIQUE UMA FONTE DE PROTOTYPE POLLUTION
 		
 		1.ADICIONE UMA VÍRGULA ANTES E UMA NOVA PROPRIEDADE
 		  DENTRO DO ARQUIVO JSON NA REQUISIÇÃO.
 		
 			, "__proto__": {"foo":"bar"}
 		
 		2.ENVIE A REQUISIÇÃO
 		
 		3.SE A RESPOSTA DA REQUISIÇÃO RESPONDER O PROTOTYPE DE ERRO SIGNIFICA
 		QUE ESTÁ VULNERÁVEL.
 		
 	3.PROCURE UM GADGET.
 		
 		1.IDENTIFIQUE UMA PROPRIEDADE QUE DESEJA ALTERAR DENTRO DO ARQUIVO JSON
 		.
 		
 	4.EXPLORAR.
 		
 		ESCALONAMENTO DE PRIVILÉGIO:
 		
 			1.USE O PROTO PARA ALTERAR A PROPRIEDADE ALVO.
 				,"__proto__":{"gadget": "payload"
 			
 				EX: ,"__proto__":{"isAdmin": "true"
 				
 		RCE:
 			1.USAR OBJETOS DO NODE JS PARA ACESSAR ARQUIVOS CONFIDENCIAIS.
 				
 				SHELL: PODENDO USAR O VALOR DESSE OBJETO PARA EXECUTAR COMANDOS.
 			
 				EX:,"__proto__":{"shell":"calc"} ESSE PROTOTYPE POLLUTION ABRIRIA UMA CALCULADORA.
