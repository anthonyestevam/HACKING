INSECURE DATA STORAGE É UMA VULNERABILIDADE QUE OCORRE QUANDO DADOS SENSÍVEIS NO APLICATIVO SÃO SALVOS SEM CRIPTOGRAFIA, COM CRIPTOGRAFIAS FRACAS COMO MD5 OU SHA-1 OU SALVOS NO DIRETÓRIO SHARED_PREFERENCES QUE É UM DIRETÓRIO QUE QUALQUER APLICATIVO PODE TER ACESSO.

	1.NAVEGUE PELO APLICATIVO, INSERINDO LOGIN, SENHA, OU QUAISQUER DADOS QUE POSSAM SER INSERIDOS NO APLICATIVO E UTILIZANDO FUNÇÕES TAMBÉM.
	
	2.ENTRE NA SHELL DO ADB, COMANDO: adb shell.
	
	3.VÁ EM data/data. COMANDO: cd data/data
	
	4.LISTE OS DIRETÓRIOS E PROCURE PELO DIRETÓRIO DO APLICATIVO.
	
	5.ENTRE NO DIRETÓRIO DO APLICATIVO. COMANDO: cd <diretorio_app>
	
	6.LISTE OS DIRETÓRIOS.
	
	7.ANALISE TODOS OS DIRETÓRIOS E ARQUIVOS, PARA CHECAR SE ALGUM DADO NO APLICATIVO FOI SALVO COMO LOGINS, DADOS RECEBIDOS DE UMA API, TOKEN, ETC... SEM CRIPTOGRAFIA OU COM CRIPTOGRAFIA FRACA 
	
	DIRETÓRIOS COMUNS PARA ANALISAR: DATABASE, SHARED_PREFS.
